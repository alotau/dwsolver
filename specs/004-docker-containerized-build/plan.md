# Implementation Plan: Containerized Build for dwsolver

**Branch**: `004-docker-containerized-build` | **Date**: 2026-03-19 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/004-docker-containerized-build/spec.md`

## Summary

dwsolver requires a C build toolchain that is non-trivial to install on macOS,
Windows, or unfamiliar Linux distributions. This plan implements a two-stage
`Dockerfile` (builder stage with `build-essential` + `automake`, runner stage
with only the binary) plus a `.dockerignore` at the repository root, and a
non-blocking `docker-build` CI job in `ci.yml`. The builder stage uses the same
`touch`-the-pregenerated-files technique already proven in Spec 003's `ci.yml`.
No source file changes are required — Spec 002 already fixed all source issues on
`master`. The abandoned `set-up-docker` branch is fully superseded.

## Technical Context

**Language/Version**: Dockerfile (no language version constraint); underlying binary is C99
**Primary Dependencies**: Docker Engine ≥ 20.10; `ubuntu:24.04` base image; `build-essential` + `automake` (builder stage only)
**Storage**: None in the image; user data accessed via volume mounts (`-v <host-dir>:/data`)
**Testing**: `docker build` + smoke test (`docker run --rm dwsolver 2>&1 | grep -q "Usage:"`) in CI `docker-build` job
**Target Platform**: Linux containers (`linux/amd64` primary; `linux/arm64` when built on ARM hosts)
**Project Type**: CLI tool containerized deployment
**Performance Goals**: Final image ≤ 200 MB (SC-002); `docker build` completes within CI 30-minute timeout
**Constraints**: No source file modifications; must use `touch` trick (no `autoreconf`); must not block required CI checks (`continue-on-error: true`)
**Scale/Scope**: Three files changed/created (`Dockerfile`, `.dockerignore`, `ci.yml`); one new CI job

## Phase 0: Research Findings

All unknowns resolved. See [research.md](research.md) for full detail.

Key decisions:

- **R1**: Base image — `ubuntu:24.04` pinned; matches `build-linux` CI environment
- **R2**: Build technique — `touch` the six pre-generated autotools files before `./configure && make`; same as `ci.yml`; `autoreconf` rejected (version mismatch, was the failure mode of `set-up-docker` branch)
- **R3**: Multi-stage build — `builder` stage with `build-essential` + `automake`; `runner` stage copies only the binary; final image ~78 MB
- **R4**: ENTRYPOINT — exec-form `ENTRYPOINT ["/usr/local/bin/dwsolver"]`; no `CMD`; all user args forwarded directly
- **R5**: Binary location — `COPY --from=builder /build/src/dwsolver /usr/local/bin/dwsolver`; no `make install` needed
- **R6**: `.dockerignore` — excludes `.git/`, `*.o`, `src/dwsolver`, configure outputs, test artefacts; mirrors `.gitignore`
- **R7**: CI smoke test — `docker run --rm dwsolver 2>&1 | grep -q "Usage:"`; `continue-on-error: true`; no volume mount required
