# Implementation Plan: Structured Test Coverage for dwsolver

**Branch**: `006-test-coverage` | **Date**: 2026-03-20 | **Spec**: [spec.md](spec.md)

## Summary

Add four layers of test coverage to the dwsolver C codebase: (1) unit tests for the pure `dw_blas.c` math functions wired into `make check`; (2) two sanitizer CI jobs (ASan/UBSan and TSan) on the Linux runner; (3) three new end-to-end example problems that exercise previously untested code paths in `phase_1_iteration`; and (4) CLI-level guidefile parsing tests. All changes are additive — no existing tests are modified or removed.

## Technical Context

**Language/Version**: C99 (GCC and Clang; MinGW-w64 on Windows)
**Primary Dependencies**: Embedded GLPK 4.44, POSIX pthreads, libc/libm
**Storage**: File-based (CPLEX `.lp` format, shell-script test runner)
**Testing**: Existing shell runner `tests/dw-tests.sh`; new `make check` target via Autotools `TESTS`; new sanitizer CI jobs
**Target Platform**: Linux (CI primary), macOS (local dev), Windows (cross-compile CI)
**Project Type**: CLI solver
**Performance Goals**: Full test suite (unit + 9 e2e examples) completes in < 2 minutes on CI
**Constraints**: No external test framework dependencies; no changes to existing test outputs; sanitizer jobs Linux-only
**Scale/Scope**: ~6 new files (test_blas.c, test_guidefile.sh, 3 example dirs, 2 workflow jobs)

## Phases

### Phase 0: Research — COMPLETE

See [research.md](research.md). All NEEDS CLARIFICATION items resolved:

- **Test harness**: hand-rolled `EXPECT_APPROX` in `tests/test_blas.c`; no framework
- **US4 scope**: CLI-level shell tests (`test_guidefile.sh`), not direct `process_cmdline` unit tests — guidefile parsing is too entangled with GLPK struct definitions
- **Sanitizer runner**: `ubuntu-latest` only; two separate jobs; `-O1` flag
- **Pre-existing UB**: `printf` format mismatch in `dw_support.c` must be fixed before sanitizer CI
- **Three new examples**: construction strategy documented per-example in research.md
- **Autotools integration**: `tests/Makefile.am` with `check_PROGRAMS` + `TESTS`
