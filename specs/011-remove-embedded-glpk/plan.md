# Implementation Plan: Remove Embedded GLPK Source

**Branch**: `011-remove-embedded-glpk` | **Date**: 2026-03-22 | **Spec**: [spec.md](spec.md)

## Summary

Drop the ~100-file verbatim GLPK 4.44 copy from `src/` and link against a
system-installed GLPK ≥ 4.65 library instead.  Three deprecated `lpx_*` call
sites must be migrated to modern `glp_*` equivalents.  The CI workflows, the
Dockerfile, the pkg-config file, and the `README.md` must all be updated in
the same change.  The `glpk_mutex` guards around GLPK file-I/O are preserved
unchanged.  The public API (`dw_solver.h`) is already free of GLPK types so no
header refactoring is required.

## Technical Context

**Language/Version**: C99  
**Primary Dependencies**: GLPK ≥ 4.65 (external, via `pkg-config`), POSIX pthreads  
**Storage**: N/A (solver reads/writes LP files from disk)  
**Testing**: Shell test runner (`tests/dw-tests.sh`); objective values compared to pre-refactoring baseline with 1e-6 relative tolerance  
**Target Platform**: macOS (Homebrew), Linux (apt/yum/dnf); Windows is out of scope for this feature  
**Project Type**: library + CLI tool  
**Performance Goals**: No change; this is a build-system refactoring only  
**Constraints**: Must preserve all `glpk_mutex` guards; must not change solver numerical behaviour; must not change `dw_solver.h` public API  
**Scale/Scope**: ~100 source files removed; 10 `lpx_*` call sites migrated; 5 build/CI files updated
