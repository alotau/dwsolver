# Contract: Source Tarball Contents

**Feature**: 012-release-infrastructure

Documents the required contents of every `dwsolver-<ver>.tar.gz` produced by
`make dist` / `make distcheck`.  This contract is enforced by `EXTRA_DIST`
entries across all `Makefile.am` files and verified by `make distcheck`
(which must exit 0 for a release to proceed).

---

## Required Top-Level Entries

All paths are relative to the `dwsolver-<ver>/` root inside the tarball.

### Automake-managed source (always included automatically)

| Path | Source `Makefile.am` | Notes |
|------|---------------------|-------|
| `src/*.c` | `src/Makefile.am` via `libdwsolver_la_SOURCES`, `dwsolver_SOURCES` | All C source files |
| `src/*.h` | `src/Makefile.am` via `include_HEADERS`, `noinst_HEADERS` | All header files |
| `configure.ac` | top-level | Autoconf input |
| `Makefile.am` | top-level | |
| `src/Makefile.am` | top-level | |
| `tests/Makefile.am` | top-level | |
| `config.h.in` | Generated by `autoheader` | |
| `dwsolver.pc.in` | `EXTRA_DIST` in top-level `Makefile.am` | pkg-config template |

| Path | Listed in | Purpose |

## Files EXCLUDED from the Tarball

The following are generated or transient and must NOT appear in the tarball:

| Pattern | Reason |
|---------|--------|
| `tests/relaxed_solution` | Written by `dw-tests.sh`; listed in `DISTCLEANFILES` |
| `tests/rs_sorted` | Written by `dw-tests.sh`; listed in `DISTCLEANFILES` |
| `tests/out_obj.txt` | Written by `dw-tests.sh`; listed in `DISTCLEANFILES` |
| `tests/src/dw_blas.Po` | GCC dependency tracking; listed in `DISTCLEANFILES` |
| `dwsolver-*.tar.gz` | The tarball itself |
| `*.o`, `*.lo`, `*.la` | Compiled objects |
| `.libs/` | Libtool scratch directory |

## Verification Command

```sh
# After make distcheck succeeds, inspect tarball contents:
tar tzf dwsolver-$(grep AC_INIT configure.ac | grep -o '[0-9]\+\.[0-9]\+\.[0-9]\+').tar.gz | sort
```

To spot-check required entries:
```sh
VERSION=$(grep -oP '(?<=AC_INIT\(dwsolver, )[^,]+' configure.ac)
tar tzf dwsolver-${VERSION}.tar.gz | grep -E \
  'examples/|architecture/|specs/|Dockerfile|\.github/|tests/dw-tests\.sh|tests/fixtures/'
```

Expected: all patterns produce at least one matching line.

---

## `make distcheck` Self-Test

`make distcheck` validates this contract automatically by:
1. Building the tarball with `make dist`
2. Unpacking into `dwsolver-<ver>/_build/sub/`
3. Running `./configure && make && make check` from the unpacked tree (any
   missing file causes a build or test failure here)
4. Running `make install && make uninstall` into a temp prefix
5. Running `make distclean` and verifying no files remain (`distcleancheck`)

A passing `make distcheck` is therefore sufficient proof that this contract
is satisfied.
