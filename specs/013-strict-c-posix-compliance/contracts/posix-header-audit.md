## `dw.h` Include-Before-Guard Note

The file `dw.h` places `<semaphore.h>`, `<fcntl.h>`, `<pthread.h>`, `<glpk.h>`,
`<stdlib.h>`, `<stdio.h>`, `<string.h>`, and `<errno.h>` **before** the
`#ifndef DECOMPOSE_H_` include guard. This is non-standard style but not a standards
violation — each system header has its own idempotency guard. With `_POSIX_C_SOURCE=200809L`
defined via `AM_CPPFLAGS` (processed by the compiler before any `#include`), the
POSIX interfaces are correctly exposed.

**Action**: Document in `acceptance-report.md`; do not reorder (risk of visibility-macro
ordering side-effects).
