## Rule 10 — ga-buffer

**Rule name**: Do not form or use out-of-bounds pointers or data
**SEI CERT C correspondence**: ARR30-C / STR31-C
**Prior work**: None (not covered by spec 008 or spec 013 checks)
**Verdict**: **FAIL → remediated (see FR-003)**

**Finding**: `strcpy(local_col_name, var_name)` at three call sites copies
from `glp_get_col_name(sub_lp, j)` into a `BUFF_SIZE`-sized (200-byte) heap
buffer. GLPK's internal name limit is 255 chars; no length check exists.

| File              | Line | var_name source          |
|-------------------|------|--------------------------|
| dw_rounding.c     | 425  | `glp_get_col_name(sub_lp, j)` |
| dw_rounding.c     | 445  | `glp_get_col_name(sub_lp, j)` |
| dw_support.c      | 668  | `glp_get_col_name(sub_lp, j)` |

**Remediation (FR-003)**: Replace with `strncpy(dst, var_name, BUFF_SIZE-1)`
followed by `dst[BUFF_SIZE-1] = '\0'`.

**Post-remediation verdict**: ✅ PASS (T007 complete — 2026-03-23).

Post-T007 cppcheck run (see `audit/cppcheck-post-remediation.txt`): zero unbounded strcpy findings at the three remediated sites. Diff vs. pre-remediation baseline shows only line-number shifts in pre-existing `nullPointerOutOfMemory` false-positives — no new finding categories introduced, no prior findings eliminated except target sites.

**Pass cases within this rule** (not requiring remediation):
- `dw_rounding.c:130` copies from master LP column names set via
  `snprintf(_, BUFF_SIZE-1, ...)` in `dw_phases.c:124-128` — provably bounded.

## Rule 15 — nonnullptr

**Rule name**: Do not pass a null pointer to a library function that expects a non-null argument
**SEI CERT C correspondence**: EXP34-C
**Prior work**: None (not addressed by spec 008 or spec 013)
**Verdict**: **FAIL → remediated (see FR-003)**

**Finding**: `strtok(NULL, ",")` return value passed directly to `strcpy()`
without a NULL check at five call sites:

| File            | Line | Pattern                                                     |
|-----------------|------|-------------------------------------------------------------|
| dw_rounding.c   | 427  | `strcpy( curr_flight, strtok(NULL, ",") )`                  |
| dw_rounding.c   | 428  | `strcpy( curr_sector, strtok(NULL, ",") )`                  |
| dw_rounding.c   | 447  | `strcpy( temp_flight, strtok(NULL, ",") )`                  |
| dw_rounding.c   | 448  | `strcpy( prev_sector, strtok(NULL, ",") )`                  |
| dw_support.c    | 673  | `strcpy( sector_name, strtok(NULL, ",") )`                  |

`strtok` returns `NULL` when no delimiter is found. `strcpy(ptr, NULL)` is
undefined behaviour (C11 §7.24.2.3, footnote 309). The existing guard
`if( strtok(local_col_name, "(") == NULL ) printf(...)` does not branch;
the body prints but then falls through to the nonnullptr-triggering `strtok`
calls.

**Remediation (FR-003)**: Extract `strtok` call to a named variable; branch
on NULL with `continue` (skip column); replace `strcpy` with `strncpy`.

**Post-remediation verdict**: ✅ PASS (T008 complete — 2026-03-23).

Post-T008 cppcheck run (see `audit/cppcheck-post-remediation.txt`): zero strtok-to-strcpy
null-dereference patterns remain at the five remediated sites. Build succeeds with
`-std=c11 -pedantic-errors`; all 9 regression tests pass (`tests/dw-tests.sh`).

## Rule 16 — nullref

**Rule name**: Do not dereference a null pointer
**SEI CERT C correspondence**: EXP34-C
**Prior work**: Spec 008 (runtime OOM protection)
**Verdict**: **PASS**

**Evidence**: All `malloc` returns are protected by `dw_oom_abort()` which
calls `exit(EXIT_FAILURE)` on NULL (see `src/dw.h:196-199`). This prevents
any subsequent null dereference from unguarded allocations.

`glp_get_col_name()` returns NULL for unnamed columns; such calls are either
passed to `%s` format (safe on POSIX) or are covered by the nonnullptr
remediation (rules 10/15 above).

Note: `cppcheck` emits `nullPointerOutOfMemory` for these sites because it
does not recognise `dw_oom_abort` as a null-terminating check. These are
false positives; they appear in the spec 013 baseline and are suppressed
with `// cppcheck-suppress nullPointerOutOfMemory` at the call site.
